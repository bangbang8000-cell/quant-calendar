## 一、产品目标

将量化选股日历的 UI/UX 从 **"功能可用但体验粗糙"（5.3/10）** 提升至 **"专业、流畅、一致"（目标 8.0/10）**。

|------|------|------|
| 综合评分 | 5.3/10 | 8.0/10 |

| 移动端体验 | 2 星 | 4 星 |

## 二、功能范围

### 2.1 阶段一：关键修复（P0/P1，2-3 天）

#### F1: 平板端侧边栏修复
- **现状**：Tablet 模式下侧边栏图标完全隐藏，导航失效
- **目标**：侧边栏折叠态保留图标，`responsive.css` 与 `themes.css` 折叠逻辑统一
- **验收**：平板端（768-1024px）侧边栏显示图标，点击可切换页面

#### F2: Composables 死代码清理
- **现状**：8 个 Composables 定义但未接入 setup()，setup() 内重复实现
- **目标**：删除未使用的 Composables，或将其接入 setup() 替换重复代码
- **验收**：所有 Composable 要么被 setup() 使用，要么删除

#### F3: Token 定义去重
- **现状**：tokens.css 和 themes.css 重复定义字体、间距、语义色
- **目标**：tokens.css 为唯一 Token 来源，themes.css 仅定义主题色变量
- **验收**：无重复 CSS 变量定义

#### F4: 移动端底部导航优化
- **现状**：底部导航栏仅显示 emoji，无文字标签
- **目标**：添加文字标签（策略/日历/评股/研究/配置），适配安全区域
- **验收**：移动端底部导航栏 icon + 文字，清晰可辨

#### F5: 字体硬编码消除
- **现状**：themes.css 中 303 处 `font-size:` 硬编码
- **目标**：100% 替换为 `var(--font-*)` 或 `var(--fs-*)` 变量引用
- **验收**：grep `font-size: [0-9]` 在 themes.css 中返回 0 结果

### 2.2 阶段二：体验升级（P2，1-2 周）

#### F6: 键盘导航
- **目标**：侧边栏菜单支持 Tab 键聚焦，Enter 键激活，Esc 关闭弹窗
- **验收**：纯键盘可完成导航和基本操作

#### F7: 面包屑导航
- **目标**：全局 Header 显示当前位置（如 策略总览 > 策略共识度）
- **验收**：面包屑正确反映当前页面层级，点击可跳转

#### F8: 骨架屏加载
- **目标**：数据加载时显示骨架屏替代空白 loading 文字
- **验收**：策略总览、日历、评股页面加载时显示骨架屏，数据到达后平滑过渡

#### F9: 操作确认对话框
- **目标**：登出、删除、重置等危险操作弹出确认框
- **验收**：危险操作触发 `ElMessageBox.confirm`，用户确认后执行

#### F10: 页面状态缓存
- **目标**：使用 `<keep-alive>` 缓存页面滚动位置和表单状态
- **验收**：切换页面后返回，滚动位置和表单数据保留

#### F11: 主题系统优化
- **目标**：合并 classic-white/red/gold 为参数化主题，减少 Element Plus 覆盖代码
- **验收**：5 套主题（tech-blue, rose-red, vibrant-orange, classic, dark-pro），classic 支持色相切换

#### F12: 主题切换过渡动画
- **目标**：`* { transition: background-color 0.3s, color 0.3s, border-color 0.3s; }`
- **验收**：切换主题时颜色平滑过渡

#### F13: 内容区最大宽度
- **目标**：`.main-content { max-width: 1400px; margin: 0 auto; }`（超宽屏居中）
- **验收**：2560px 宽屏下内容区最大 1400px 居中，两侧留白

### 2.3 阶段三：移动端增强（P3，3-5 天）

#### F14: 触摸手势
- **目标**：移动端支持左右滑动切换 Tab 页面
- **验收**：触摸滑动切换页面，有视觉过渡

#### F15: 表格横向滚动优化
- **目标**：移动端表格添加滚动阴影指示器，`-webkit-overflow-scrolling: touch`
- **验收**：移动端表格可流畅横向滚动，两侧有渐变阴影指示

#### F16: 顶部安全区域适配
- **目标**：`padding-top: env(safe-area-inset-top)` 适配刘海屏
- **验收**：iPhone 刘海屏顶部内容不被遮挡

## 三、非功能需求

### NFR1: 零回归
- 所有修改不得破坏现有业务功能
- 每完成一个 Feature 需要手动验证：登录、策略总览、日历、评股、研究、配置

### NFR2: 性能
- 首屏加载时间不增加（CSS 变量引用不增加渲染开销）
- 主题切换延迟 < 100ms

### NFR3: 兼容性
- 桌面端：Chrome 90+, Edge 90+, Firefox 90+
- 移动端：iOS Safari 14+, Android Chrome 90+
- 暗色主题通过 WCAG AA 对比度标准（≥ 4.5:1 正文，≥ 3:1 大文本）

### NFR4: 代码质量
- CSS 文件总行数不超过 5000 行（当前 4456 + 88 + 93 + 166 + 191 = 4994 行）
- index.html 中 setup() 函数不超过 2000 行（当前 ~4400 行）
- 无内联 style（当前 41 处 `style="..."`）

## 四、用户故事

### US1: 平板用户导航
> 作为平板用户，我希望侧边栏折叠后仍能看到导航图标，以便我能快速切换功能模块。

### US2: 移动端新用户
> 作为移动端新用户，我希望底部导航栏有清晰的文字标签，以便我能理解每个图标的功能。

### US3: 暗色主题用户
> 作为暗色主题用户，我希望切换主题时颜色平滑过渡，而不是突变闪烁。

### US4: 深度操作用户
> 作为管理员，我在执行删除、登出等操作时，希望有确认对话框防止误操作。

### US5: 键盘操作用户
> 作为习惯键盘操作的用户，我希望能用 Tab 键和 Enter 键完成导航，不需要鼠标。

## 五、风险与假设

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| setup() 重构引入 bug | 中 | 高 | 逐模块迁移，每步手动验证 |
| CSS 变量替换遗漏 | 中 | 中 | grep 扫描验证，保留备份 |
| `<keep-alive>` 与 Transition 冲突 | 低 | 中 | 先测试单页面，确认兼容后推广 |
| 主题合并导致视觉差异 | 低 | 低 | 逐主题对比截图 |

**假设**：
- 前端无需引入新的 npm 依赖（CDN 模式下所有功能使用原生 CSS/JS 实现）
- 后端 API 无需修改
- 当前 7 套主题的视觉风格已得到用户认可，优化仅限代码层面
