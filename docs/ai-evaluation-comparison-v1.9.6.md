## 二、最值得借鉴的6个特性

### 1. 决策仪表盘 (Decision Dashboard) ⭐⭐⭐

**我们的**: 9维度加权分→5档推荐→简单建议  
**DSA的**: 五大模块决策仪表盘

```
dashboard:
  core_conclusion    → 一句话结论 + 信号类型(🟢🔴⚠️) + 时效性 + 分持仓建议
  data_perspective   → 趋势/价格位/量能/筹码四维透视
  intelligence       → 新闻/风险/利好/业绩/舆情
  battle_plan        → 狙击点(买入价×2+止损+止盈) + 仓位策略 + ✅⚠️❌检查清单
  phase_decision     → 阶段约束 + 行动窗口 + 观察条件
  signal_attribution → 技术/新闻/基本面/市场环境贡献度 + 最强信号
```

前端展示为结构化表格(颜色编码) + 信号卡片，信息密度高但可读性好。

### 2. 决策稳定性后校准 ⭐⭐⭐

LLM输出后自动校准，防止单日涨跌导致频繁切换：

| LLM输出 | 检查条件 | 校准结果 |
|---------|---------|---------|
| buy | 接近压力位 + 资金不流入 | → hold |
| buy | 资金流出 + 未突破 | → hold |
| sell | 接近支撑位 + 资金未流出 + 无重大风险 | → hold |
| buy/sell | 资金流数据缺失 | → hold,标注资金面未确认 |

### 3. 多策略并行分析 ⭐⭐

前端以checkbox形式呈现16种策略，用户可多选组合：
- 趋势类: 默认多头趋势、均线金叉、缠论、波浪理论
- 量价类: 放量突破、缩量回踩、底部放量、箱体震荡
- 题材/事件: 热点题材、事件驱动、龙头策略
- 基本面: 成长质量、预期重估
- 情绪类: 情绪周期、一阳夹三阴

每个策略有独立YAML定义，含评分调整建议(如"多头排列+12分")。

### 4. 数据质量感知 ⭐⭐

每个数据块带8种状态标记:
```
AVAILABLE / PARTIAL / ESTIMATED / NOT_SUPPORTED / 
FALLBACK / STALE / MISSING / FETCH_FAILED
```

LLM被明确要求: **状态为stale/fallback/missing时,confidence_level不得为"高"**

数据质量权重: quote(25%) + daily_bars(25%) + technical(25%) + news(10%) + fundamentals(10%) + chip(5%)

### 5. 信号归因 + 生命周期管理 ⭐⭐

- **信号归因**: 技术指标/新闻/基本面/市场环境各自的贡献度 + 最强看多/看空信号名
- **生命周期**: 信号创建→查询→过期→失效→归档,对立信号自动失效
- **信号时间线**: 30/90/180天信号变化追溯

### 6. 四阶段分析进度可视化 ⭐

前端展示分析进度: K线数据→筹码分析→情报搜索→综合研判,每阶段完成有反馈。

## 四、实施建议(按难度排序)

### 立即可行(改prompt,不改架构)

1. **Prompt增强** — 增加狙击点/仓位建议/信号归因字段(向后兼容)
2. **数据质量标记** — 在prompt中注明数据时间戳和来源状态
3. **防抖逻辑** — 连续评估时要求LLM解释大幅波动

### 短期(小重构)

4. **市场阶段感知** — 前端传当前时间,后端注入阶段上下文到prompt
5. **决策后校准** — 对LLM输出的buy/sell做支撑/压力位校验
6. **JSON完整性保障** — 必填字段检测+重试+占位回退

### 中期(中等改动)

7. **多策略选择UI** — 前端加策略checkbox,后端按策略组合prompt
8. **报告模板化** — Jinja2渲染,多格式输出

### 长期(大项目)

9. **LiteLLM迁移** — 替换自建provider系统,获得cross-model fallback+prompt cache
10. **多Agent管道** — Technical→Intel→Risk→Decision Agent协作
