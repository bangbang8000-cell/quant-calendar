## 1. 产品定位

量化选股日历是一个面向 A 股量化投资者的决策辅助工具，基于美林时钟宏观经济周期理论，整合多策略选股信号、AI 评估、回测分析和数据可视化。

### 1.1 核心价值主张

| 能力层 | 描述 |
|--------|------|
| **感知层** | 美林时钟五维度宏观评分，实时识别经济周期阶段 |
| **信号层** | 多因子策略选股，共识榜 + 股票池 |
| **决策层** | AI 多模型串行评估 + 信号归因 + 狙击点 + 仓位建议 |
| **验证层** | 回测引擎 + 评估历史对比 + 操作检查清单 |
| **协作层** | 飞书定时推送 + 多用户权限管理 |

### 1.2 用户

当前用户：独立量化交易者，单用户使用，主要场景为每日选股信号查看和 AI 辅助评估。

## 3. 开发安全铁律

> **v2.2.0/v2.3.0 失败教训，后续所有版本强制执行。**

### 3.1 前端修改红线

| 规则 | 说明 |
|------|------|
| **禁止批量正则替换 HTML** | Python re.sub 可能匹配 script 块内代码，破坏 JS 语法 |
| **禁止跨版本大规模重构** | 样式迁移 + toast 降噪 + Vue SFC 不得合并在同一版本 |
| **每次改动后必须浏览器验证** | 登录页 + 策略总览 + 日历视图 三项核心功能必须通过 |
| **CDN 版本始终保持可用** | 任何改动不得破坏 CDN index.html 的渲染 |
| **CSS 类名不与 CDN 内联 style 冲突** | 新增 CSS 规则必须确认 CDN 版本不受影响 |

### 3.2 通用安全规则

| 规则 | 说明 |
|------|------|
| **每版本只做一件事** | 安全加固 OR 样式迁移 OR 组件化，不混合 |
| **必须先提交 working tree** | 开始任何修改前 `git stash` + 确认 clean |
| **数据文件不受 git 保护** | `data/*.json` 在操作前必须手动备份 |
| **stash pop 后检查冲突标记** | `grep '<<<<<<<\|>>>>>>>'` 确认零冲突 |
| **发布前必须 CDN 版本冒烟测试** | 登录 → 日历 → AI评估 → 回测 四项全部通过 |
| **每次改动 ≤ 50 行** | 超过则拆分为多次 commit，便于 `git bisect` |

### 3.3 版本发布 Checklist

- [ ] `python -m pytest tests/ -q` 全部通过
- [ ] CDN 版本浏览器冒烟（登录、日历、AI评估、回测）
- [ ] `grep '<<<<<<<\|>>>>>>>'` 零冲突标记
- [ ] `git status` 确认无未提交变更
- [ ] 数据文件 `data/*.json` 已备份
- [ ] tag 创建后验证 `git show vX.Y.Z` 内容正确
