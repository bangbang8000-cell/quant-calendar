# 量化选股日历 v2.X.X — 开发路线图 (修订版)

> **基线**: v2.1.0 | **修订日期**: 2026-07-15 | **修订原因**: v2.2.0/v2.3.0 失败教训

---

## 核心原则

1. **每版本只做一件事** — 安全/后端/前端/架构严格分开
2. **小步提交，随时可回滚** — 每次改动 ≤ 50 行，每改必验证
3. **CDN 版本零影响** — 前端改动不得破坏现有功能
4. **数据文件先备份** — `data/*.json` 不受 git 保护

## 路线概览

```
v2.1.0 ✅             v2.2.0                v2.3.0                v2.4.0
[稳定加固]            [后端增强]            [前端体验]            [架构升级]
    │                     │                     │                     │
 安全+测试+清理       AI流式+策略+缓存     Toast+移动端+样式     Vue SFC迁移
```

## v2.4.0 — Vue SFC 迁移（独立版本）

**前置条件**: v2.3.0 稳定运行 ≥ 2 周，零回归 bug

**时间**: ~6 周 | **优先级**: 🟡 | **安全级别**: 低（高风险操作，严格防护）

### 四阶段迁移

**阶段 1: 并行环境（不影响 CDN）**
- Vite 构建环境，`frontend/src/` 目录
- CDN index.html 完全不动
- Vite dev server 独立验证

**阶段 2: 逐组件迁移**
- 一次只拆一个组件
- 拆分后立即在两环境对比验证
- 每个组件迁移后 `git commit`

**阶段 3: 切换**
- 全部完成后，生产入口切到 dist/
- CDN 版本保留为 fallback ≥ 1 个大版本

**阶段 4: 清理**
- 确认 Vite 版本稳定 2 周后移除 CDN 文件

## 风险管控

| 风险 | 缓解 |
|------|------|
| 前端改动导致 SPA 蓝屏 | ≤50行/次，每次浏览器验证，随时 git revert |
| 数据文件丢失 | 操作前 `cp -r data/ data.backup/` |
| CSS 冲突 CDN 样式 | CSS 改动先检查不影响 CDN 版本 |
| git 操作遗留冲突标记 | 每次操作后 `grep '<<<<<<<' ` 确认 |
